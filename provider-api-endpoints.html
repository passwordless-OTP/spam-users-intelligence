Abstract API

Endpoint: https://emailvalidation.abstractapi.com/v1/

Method: GET

Parameters:

  • api_key (required): Your Abstract API key

Kickbox

Endpoint: https://api.kickbox.com/v2/verify

Method: GET

Parameters:

  • email (required): The email address to verify
  • apikey (required): Your Kickbox API key
  • timeout (optional): Maximum time, in seconds, for the verification to complete (1-10)

Example Request:

GET https://api.kickbox.com/v2/verify?email=test@example.com&apikey=YOUR_API_KEY

Example Response:

{
  "result": "deliverable",
  "reason": "accepted_email",
  "role": false,

Our API Wrapper Endpoint

Our service provides a unified API endpoint that abstracts away the differences between these providers:

Endpoint: /api/v1/email/validate

Method: POST

Headers:

  • Content-Type: application/json
  • Authorization: Bearer YOUR_API_TOKEN

Request Body:

{
  "email": "test@example.com",
  "preferred_provider": "abstract_api"  // Optional
}

Example Response:

{
  "success": true,
  "data": {
    "email": "test@example.com",
    "is_valid": true,
    "is_disposable": false,
    "is_role_based": false,
    "domain_score": 0.85,
    "fraud_score": 0.1,
    "provider": "abstract_api",
    "source": "api_lookup"
  }
}

Usage Statistics Endpoint

To retrieve usage statistics for billing purposes:

Endpoint: /api/v1/email/usage

Method: GET

Headers:

  • Authorization: Bearer YOUR_API_TOKEN

Example Response:

{
  "success": true,
  "data": {
    "lookup_count": 1250,
    "billing_cycle_start": "2025-04-01T00:00:00Z",
    "billing_cycle_end": "2025-04-30T23:59:59Z"
  }
}
